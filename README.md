# THM---FlareVM-Arsenal-of-Tools
סקריפט הזה לקח מערכת Windows רגילה והפך אותה למעבדת Malware Analysis ו-DFIR (Digital Forensics and Incident Response) מהמתקדמות בעולם.
<img width="1723" height="514" alt="image" src="https://github.com/user-attachments/assets/1d61600f-42cf-434c-bb3e-41feb641f9de" />




FlareVM (ראשי תיבות של Forensics, Logic Analysis, and Reverse Engineering) היא אסופה מקיפה ומוקפדת של כלים ייעודיים, שנבנתה במיוחד כדי לענות על הצרכים של מהנדסים לאחור (Reverse Engineers), אנליסטים של נוזקות, מגיבי אירועים (Incident Responders), חוקרים פורנזיים ובודקי חדירות.

ערכת הכלים הזו, שפותחה במומחיות על ידי צוות ה-FLARE המפורסם של חברת FireEye (כיום Mandiant), מהווה כלי עזר אדיר לפענוח תעלומות דיגיטליות, הבנת התנהגות של נוזקות וצלילה לפרטים המורכבים ביותר של קבצי הרצה (Executables).





🗂️ מפת הדרכים של FlareVM: הכלים המרכזיים
1. הנדסה לאחור ודיבאגינג (Reverse Engineering & Debugging)
קטגוריה זו נועדה ל"פירוק" הנוזקה לחתיכות והבנה של הלוגיקה הפנימית שלה (קוד האסמבלי).

Ghidra: כלי קוד פתוח רב-עוצמה שפותח על ידי ה-NSA ומאפשר להפוך קוד מכונה בחזרה לפסאודו-קוד קריא (Decompiler).

x64dbg: הדיבאגר המוביל כיום בשוק לניתוח דינמי של קבצי 32-bit ו-64-bit בשורות קוד בזמן אמת.

2. ניתוח סטטי ודינמי (Static & Dynamic Analysis)
כלים להבנת הקובץ לפני הרצתו (סטטי) ומעקב אחר ההתנהגות שלו במערכת בזמן הריצה (דינמי).

Process Hacker: כלי מתקדם לצפייה בתהליכים רצים ומניפולציה של הזיכרון שלהם (גרסה משופרת בהרבה של מנהל המשימות הרגיל).

PEview / Detect It Easy (DIE): כלים לניתוח המבנה של קבצי הרצה (PE) וזיהוי האם התוקף ניסה "לדחוס" או להצפין את הנוזקה (Packers/Cryptors).

3. חקירת קבצים ורשת (File & Network Analysis)
HxD: עורך הקסדצימלי (Hex Editor) קליל ומהיר המאפשר לנו לפתוח כל קובץ בינארי, לראות את ה-Raw Data שלו ולערוך אותו.

Wireshark: מנתח התעבורה המוכר, שמשמש אותנו לראות אילו בקשות רשת הנוזקה מייצרת.

4. חבילת Sysinternals (מבית מיקרוסופט)
כלים קריטיים לכל איש סיסטם ואנליסט סייבר ב-Windows:

Process Monitor (ProcMon): כלי שמקליט בזמן אמת כל שינוי בקבצים, ברג'יסטרי (Registry) וברשת.

Autoruns: מציג לנו בדיוק אילו תוכנות וקבצים מוגדרים לעלות אוטומטית עם הדלקת המחשב (כדי לראות איפה הנוזקה ניסתה להשיג שרידות - Persistence).
